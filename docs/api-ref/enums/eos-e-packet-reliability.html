EOS_EPacketReliability

EOS API reference page for EOS_EPacketReliability

1 min to read

Google has informed developers of a vulnerability in versions (earlier than M102) of WebRTC. Impacts, workarounds, and updates can be found here.

This enumerated type is part of the P2P Interface.

Remarks

Types of packet reliability. Ordered packets will only be ordered relative to other ordered packets. Reliable/unreliable and ordered/unordered communication can be sent on the same Socket ID and Channel.

Values

EOS_EPacketReliability

Enumerated Value Meaning
EOS_PR_UnreliableUnordered Packets will only be sent once and may be received out of order
EOS_PR_ReliableUnordered Packets may be sent multiple times and may be received out of order
EOS_PR_ReliableOrdered Packets may be sent multiple times and will be received in order
