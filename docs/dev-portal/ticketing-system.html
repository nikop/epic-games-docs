Ticketing System (Player Ticketing)

Provides support for your product's users through an automatically generated, preassigned email address and public web API key.

5 mins to read

Epic Online Services (EOS) provides a ticketing system for you to manage support for users of your products (games or other software applications). The EOS ticketing system, known as "Player Ticketing", provides a centralized location for your user support: it provides an automatically generated, pre-assigned email address and a public web API key.

You manage the ticketing system in the Player Ticketing page of the Developer Portal. It provides you with the following functions:

  • You can tracking open, in-progress, and closed tickets.
  • You can merging related tickets.
  • You can reply to users who have submitted a ticket and want to receive replies.

Set Up The Ticketing System With Your Product

To manage player ticketing for your product, go to the your product's Game Services page, scroll down the page and click Player Ticketing.

To do this:

  1. Go to your product page:
    • In the Developer Portal (dev.epicgames.com/portal), from the left navigation, under Your Products, click the name of your product. (Note: You might have to click Back to organization before you can do this.)
  2. Once you are in your product page, from the left navigation, click Game Services.
  3. Scroll down the left navigation under Game Services and click Player Ticketing.
  4. Click Integrate APIs.
    Note: The first time you use player ticketing for a product, there is an Integrate APIs button. You must click Integrate APIs before you can use player ticketing with your product. You only need to do this once for a product. After you have clicked it, the Developer Portal replaces it with Show API key.
  5. Click Show API key and follow the onscreen instructions to view the public web API key for the product.
    You use this key to implement the API. Each product has its own public web API key, but the ticketing system uses the same email address for all your products.

Note: The Player Ticketing page in the Developer Portal is also where you see your user's support tickets. You use this page to track and manage the tickets.

How to Use the API

The ticketing system sends emails from your end-users to your dedicated mailbox. To trigger this process do the following:

  • Send a POST request to https://dev.epicgames.com/portal/api/v1/services/tickets/submit.
  • Include your product's API key in the HTTP authorization field and the information that you want to report.

The ticketing system supports the following fields:

Field Name Required Description
subject Yes One of the following:
  • open-question
  • technical-support
message Yes The email text, written by the user.
sender_email Yes The user's email address.
sender_name No The user's name.
guid No Your game's unique identifier for the user (for example, the productUserId). Note: Any type of unique identifier works, there are no restrictions enforced regarding the guid.
error_code No An error code for technical support emails.
system_os No The user's operating system.
system_antimalware No The user's anti-malware system.
system_other No Other information about the user's system.

Export User Data

To export a user’s data, for example to fulfill a GDPR request, do the following actions:

  • Send a GET request to https://dev.epicgames.com/portal/api/v1/services/tickets/datarights/{email} (where {email} is a user's email address).
  • Include your product's API key in the HTTP authorization field.
  • In the URL path, include the email address for the user you wish to export the data for.

Usage Patterns, Restrictions, and Limitations

When you make requests through the ticketing system, observe the following limitations:

  • Requests must come from websites or the product client (game client).
  • All requests must contain written material from the user.
  • The number of requests produced in a day must not exceed 10,000, or 5 per 100 Daily Active Users.
  • User data in the ticketing system is separate from user data associated with your product that you manage through EOS Player Search. You cannot export or delete user data in the ticketing system through EOS Player Search. (For information on EOS Player Search, see documentation: EOS Player Search.)
