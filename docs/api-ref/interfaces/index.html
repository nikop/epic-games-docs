EOS API Interface List

List of all interfaces in the EOS API

11 mins to read

Achievements Interface

Use the Achievements Interface to do the following:

  • Retrieve data about the state of a specific player's achievements.
  • Unlock achievements manually via calls.
  • Retrieve data about all of the achievements belonging to an application.

Further information:

  • See the Achievements Interface API Reference for more information on the APIs.
  • See the Achievements Interface documentation for more information on the Achievements Interface.

Anti-Cheat Interfaces

You can use the Anti-Cheat interfaces to help reduce cheating in online multiplayer play. They provide tools to start your game client in an anti-cheat protected mode which is then validated by either a game server (using the Anti-Cheat Server Interface) or another game client (using the Anti-Cheat Client Interface). These interfaces are not digital rights management (DRM) or code obfuscation tools and are not useful for offline or solo play.

Further information:

  • See the Anti-Cheat Interface API references for more information on the APIs:
    • See the Anti-Cheat Server Interface API Reference for more information on the Anti-Cheat Server APIs.
    • See the Anti-Cheat Client Interface API Reference for more information on the Anti-Cheat Client APIs.
  • See the Anti-Cheat Interfaces documentation for more information on the Anti-Cheat interfaces.

Auth Interface

You can use the Auth Interface to let players sign in to their Epic Games account directly from your game. Use the interface to authenticate players, get access tokens, and handle other account-related interactions with Epic Online Services. When a player signs in to their Epic Games account, you can give them access to other Epic Account Services such as Friends, Presence, User Info and Ecom Interfaces.

Further information:

  • See the Auth Interface API Reference for more information on the APIs.
  • See the Auth Interface documentation for more information on the Auth Interface.

Connect Interface

You can use the Connect Interface to let players sign in to your game through a supported identity provider.

The Connect Interface:

  • Provides access to the Epic Online Services (EOS) Game Services.
  • Generates a common unique player identifier within the cross-platform services.
  • Links external identity providers' accounts to our services.

Further information:

  • See the Connect Interface API Reference for more information on the APIs.
  • See the Connect Interface documentation for more information on the Connect Interface.

Custom Invites Interface

The Custom Invites interface is designed to allow titles to use the built-in Epic Online Services (EOS) Connect messaging to send and receive invites with an arbitrary payload. This should be considered the only viable invite interface for titles which already have an existing, robust invitation system and want players to be able to integrate sending and receiving those invites through the EOS Social Overlay UI.

Further information:

  • See the Custom Invites Interface API Reference for more information on the APIs.
  • See the Custom Invites Interface documentation for more information on the Custom Invites Interface.

Ecom Interface

The Ecom Interface gives developers using Epic Online Services the ability to interact with the Epic Games Store. With this interface, you can manage products ranging from full games and downloadable content (DLC) to virtual goods and in-game currency. This includes making offers, completing purchase transactions, verifying ownership, and redeeming purchased items.

Further information:

  • See the Ecom Interface API Reference for more information on the APIs.
  • See the Ecom Interface documentation for more information on the Ecom Interface.

Friends Interface

Playing games with your friends and meeting new players online are important parts of many online services. The Epic Online Services (EOS) SDK uses the Friends Interface to retrieve the friends list for a signed-in player.

Friends lists are stored by the online service's servers. Friends lists can change during a session as friends are added or removed or if friends grant or revoke consent for the game to use their information.

Upon successful completion of a friends list query, the Friends Interface creates a local cache that is used by all other Friends Interface functions. Additionally the SDK receives notifications from other Epic Games services about events that lead to mutations of the friends list, such as a friend being removed, an invite being accepted or a friend revoking their consent to have their information used by the game.

Further information:

  • See the Friends Interface API Reference for more information on the APIs.
  • See the Friends Interface documentation for more information on the Friends Interface.

Integrated Platform Interface

If you want to release your game with Epic Online Services (EOS) on multiple platforms and allow players to use the Social Overlay across these platforms, then you need to use the Integrated Platform API and register a number of callbacks when you initialize the EOS SDK. This allows players to use the Social Overlay across platforms to invite other players to join their game or allow other players to request to join their game.

Further information:

  • See the Integrated Platform Interface API Reference for more information on the APIs.
  • See the documentation for more information on native platform integration:
    • See the Native Platform Integration documentation for more information on the Platform Interface.
    • See the How to Use the Social Overlay for Crossplay documentation for more information on how to integrate the Social Overlay into your game.

KWS Interface

This interface is not available for general access at this time. Instead, you can integrate Kids Web Services into your game using a dedicated API that is separate from the EOS SDK.

Further information:

  • See the Kids Web Services website at www.kidswebservices.com for more information.

Leaderboards Interface

Use the Leaderboards Interface to rank scores from your entire player base, so that players can compete with their friends or other players worldwide for the top score. Each game using EOS leaderboards can support multiple leaderboards, collecting scores from different sources, and ranking them with different scoring modes.

Further information:

  • See the Leaderboards Interface API Reference for more information on the APIs.
  • See the Leaderboards Interface documentation for more information on the Leaderboards Interface.

Lobby Interface

Lobbies provide a persistent connection between players for the purpose of sharing game and user states with real-time updates. Typically, players can create or join lobbies to form teams, select pre-game options, and wait for additional players to join in before playing together. Using the Lobby Interface, your players can create, join, leave, and manage lobbies.

Further information:

  • See the Lobby Interface API Reference for more information on the APIs.
  • See the Lobby Interface documentation for more information on the Lobby Interface.

Metrics Interface

The Metrics Interface tracks your game's usage and populates the Game Analytics dashboard in the Developer Portal. This data includes active, online instances of the game's client and server, and past sessions played by local players. To access the Game Analytics dashboard, sign in to the Developer Portal at dev.epicgames.com/portal, and select Analytics in the left-hand navigation.

Further information:

  • See the Metrics Interface API Reference for more information on the APIs.
  • See the Metrics Interface documentation for more information on the Metrics Interface.

Mods Interface

Use the Epic Online Services (EOS) Mods Interface to manage mods. You can only use this interface to manage mods that are on the Mod Marketplace of the Epic Games Store.

Further information:

  • See the Mods Interface API Reference for more information on the APIs.
  • See the Mods Interface documentation for more information on the Mods Interface.

P2P Interface

The P2P Interface enables games implementing the Epic Online Services (EOS) SDK to set up and manage peer-to-peer (P2P) connections between players. This enables them to send and receive data between one another directly, typically for the purpose of a multiplayer game. Connections made with the EOS P2P Interface are only established between authenticated players, and are secure-by-default using DTLS, which provides two distinct advantages. First, the speed of handling peer-to-peer connections is significantly increased, in that EOS's authentication greatly reduces the need for connections to be re-negotiated. Second, the process of securely handling connections is itself greatly simplified for the SDK user, abstracting out the need for detailed network socket management and condensing most functions to what data needs to be sent and to whom.

Further information:

  • See the P2P Interface API Reference for more information on the APIs.
  • See the P2P Interface documentation for more information on the P2P Interface.

Player Data Storage Interface

The Player Data Storage Interface enables developers using Epic Online Services (EOS) to save encrypted, player-specific, game-specific data to cloud servers. Data that you store through this interface is accessible to the player on any device where they can sign in. The Player Data Storage Interface supports any file format; typical use cases would include saved games and replay data.

Further information:

  • See the Player Data Storage Interface API Reference for more information on the APIs.
  • See the Player Data Storage Interface documentation for more information on the Player Data Storage Interface.

Presence Interface

With the Presence Interface, your game can share a local player's status, known as presence , and query the presence of other players online. A game can also share transient data of the player with others, in order to share more detailed information about the state of the local player. Players can only receive presence information about other players with whom they are friends.

Further information:

  • See the Presence Interface API Reference for more information on the APIs.
  • See the Presence Interface documentation for more information on the Presence Interface.

Progression Snapshot Interface

The Progression Snapshot Interface allows storing player-specific game data for the purposes of supporting an end-user experience for Epic Games account merging. As of today, the concept of merging two separate Epic Games accounts, owned by the same player, into a single Epic Games account does not exist. However, this type of account merging is a feature that will be made available to players with Epic Games accounts in the future.

The progression snapshot feature becomes relevant for players in cases where they have two separate Epic Games accounts, and have played the same game on both of the accounts. In such a case, if the player chooses to merge their Epic Games accounts into a single account, the EOS Overlay will be able to present a snapshot view of their game progress for both accounts. This allows players to choose their preferred game progression to preserve as a result of the account merge operation.

Further information:

  • See the Progression Snapshot Interface API Reference for more information on the APIs.
  • See the Progression Snapshot Interface documentation for more information on the Progression Snapshot Interface.

Reports Interface

The Reports Interface stores in-game player behavior reports. You can query these player reports from your Developer Portal to view the details of reported player behaviors, for example, cheating, verbal abuse, offensive profiles, and so on. This interface provides information that you can then use with the Sanctions Interface to conduct any punitive actions.

Further information:

  • See the Reports Interface API Reference for more information on the APIs.
  • See the Reports Interface documentation for more information on the Reports Interface.
  • See the Sanctions Interface documentation for more information on the Sanctions Interface.

Real-Time Communication (RTC) Interface

The Real-Time Communication (RTC) Interface is used to manage voice chat rooms.

Further information:

  • See the Real-Time Communication (RTC) Interface API Reference for more information on the APIs.
  • See the Real-Time Communication (RTC) Interface documentation for more information on the RTC Interface.

Sanctions Interface

The Sanctions Interface manages punitive actions taken against your players. Actions might include temporary or permanent bans from gameplay or communication bans that limit the social aspects of your product for a particular player. You define the disciplinary actions for your product to handle negative behavior based on your use cases.

Further information:

  • See the Sanctions Interface API Reference for more information on the APIs.
  • See the Sanctions Interface documentation for more information on the Sanctions Interface.

Sessions Interface

Epic Online Services (EOS) gives players the ability to host, find, and interact with online gaming sessions through the Sessions Interface. A session can be short, for example, filling a certain number of player slots before starting a game, then disbanding after the game ends. It can also be longer, for example, keeping track of a game that cycles through matches on multiple maps or levels. The Sessions Interface also manages game-specific data that supports the EOS service searching and matchmaking functionality.

Further information:

  • See the Sessions Interface API Reference for more information on the APIs.
  • See the Sessions Interface documentation for more information on the Sessions Interface.

Stats Interface

The Stats Interface provides the ability for developers to manage players' stats for an application, which can include any statistical data that a developer wishes to track, such as the number of items collected, the player's fastest completion time for a level, the total number of victories or losses, or the number of times that a player has performed a certain action. You can use stats to determine when to unlock achievements and how to rank players in leaderboards.

Further information:

  • See the Stats Interface API Reference for more information on the APIs.
  • See the Stats Interface documentation for more information on the Stats Interface.
  • See the Achievements Interface documentation for more information on the Achievements Interface.
  • See the Leaderboards Interface documentation for more information on the Leaderboards Interface.

Title Storage Interface

The Title Storage Interface enables developers using Epic Online Services (EOS) to retrieve encrypted data from cloud servers. Data that you store through this interface is accessible to any player on any device where they can log in. While similar to the Player Data Storage Interface, this interface is specialized to handle game-specific data rather than player-specific data, and can provide different versions of files based on the player's platform, region, or other conditions.

Further information:

  • See the Title Storage Interface API Reference for more information on the APIs.
  • See the Title Storage Interface documentation for more information on the Title Storage Interface.
  • See the Differences from Player Data Storage section of the Title Storage Interface documentation for more specific information about the differences between the Title Storage Interface and the Player Data Storage Interface.

UI Interface

Epic Online Services (EOS) supports product-independent overlay windows through the EOS Overlay. The UI Interface is used to access the EOS Overlay UI and its components. The overlays give players product-independent access to various features. The UI Interface manages interactions with the overlays by providing status updates, showing or hiding the overlays, and adjusting display and hotkey preferences.

Further information:

  • See the UI Interface API Reference for more information on the APIs.
  • See the UI Interface documentation for more information on the UI Interface.
  • See the EOS Overlay Overview documentation for more information on the EOS Overlay.

User Info Interface

Each Epic Online Services (EOS) player account has a unique identifier that the service uses internally to refer to the account. The User Info Interface bridges the gap between the player's account identifier and information about the player, such as display name, country and preferred language, and so on. You can retrieve this information for both remote players and signed-in, local players.

Further information:

  • See the User Info Interface API Reference for more information on the APIs.
  • See the User Info Interface documentation for more information on the User Info Interface.
