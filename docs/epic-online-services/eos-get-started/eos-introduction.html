EOS Introduction

Key information when integrating Epic Online Services (EOS) into your game.

4 mins to read

Use the Epic Online Services (EOS) SDK to integrate the following services into your game:

  • EOS Game Services
  • EOS Epic Account Services
  • Epic Game Store Services

Integrate these services into a game running on multiple platforms, including Windows, macOS, Linux, PlayStation, Xbox, and Switch. (Note you can only use Epic Games Store Services on Windows and macOS.)
For a full list of platforms and platform versions supported, see the Platforms Supported documentation.

For an overview of the features available with each service, see the following documentation: Epic Developer Resources: Overview.

EOS SDK

Interfaces

You integrate EOS services with your game through interfaces. These interfaces are SDK APIs and Web APIs. To integrate EOS services, you must call these APIs in your game’s (or software application’s) code.

The interfaces are modular and specific to each EOS service. For example:

  • The Lobbies Interface integrates your game with the EOS lobbies service. This service manages the creation and interactions of your game’s lobbies. (Documentation: Lobbies Interface.)

  • The Friends Interface integrates your game with the EOS friends service. This service manages the features associated with a player’s friends list. (Documentation: Friends Interface.)

EOS Game Services Interfaces

The interfaces in EOS Game Services consist of the following:

  • Multiplayer: lobbies, p2p, sessions, stats, title storage, voice chat
  • Player progression: metrics, stats, achievements, leaderboards
  • Game operations: anti-cheat, player sanctions, reports, ticketing for community-management

EOS Epic Account Services Interfaces

The interfaces in EOS Epic Account Services consist of the following:

  • Crossplay: Enable players accessing your game from different storefronts to play together using combined friends lists.
  • Friends: Enable players to invite friends to join a game-play session. Retrieve and display friend information to players.
  • Presence: Display a player’s status (online/offline/playing) in the game.
  • Social Overlay: Display to a player their achievements and friends information, including if their friends are playing the same game.

Platform Interface

To use any EOS services, you must integrate the Platform Interface into your game.

The Platform Interface does not provide a service, instead it facilitates your game’s integration with all other EOS services. EOS requires the Platform Interface as the first parameter to every API call that uses an interface. (Documentation: Platform Interface.)

Notes:

  • The Platform Interface is an EOS Game Services interface. However both EOS Game Services and EOS Epic Account Services require the Platform Interface.
  • "Platform" in "Platform Interface" is the EOS platform. It does not relate to the games platform your game clients run on.

Coding Languages and Game Platforms

The EOS SDK is available in C, and also has a C# wrapper.

By focusing on C, we can provide stable application binary interfaces (ABI) to ensure integration functionality, regardless of compiler choice, implementation, calling conventions, data sizes, or alignments.

This also allows independent code patching for the SDK to limit interruptions in your own development, and helps maintain the life of the SDK without recompiling the application after distributing fixes and updates. Additionally, focusing on C APIs enforces a more deliberate exposure of the public API.

There are different types of SDK download to use, depending on the game platform your game clients run on:

  • Windows, macOS, Linux - the EOS SDK in C, and the EOS SDK in C#.
  • Mobile - the EOS SDK for iOS, and the EOS SDK for Android.
  • Consoles - there are EOS SDK downloads for game consoles in C, and in C#.
    The SDK downloads for consoles are only available to developers approved by the platform holders and approved by Epic Games.
    The platform holders are: Microsoft (Xbox One, Xbox Series X), Sony (PlayStation 4, PlayStation 5), Nintendo (Switch).
    To get access to the SDK downloads for consoles and associated documentation, you must apply for platform-holder approval. You can do this once you have set up an Epic Games account and organization in the Developer Portal. See the EOS Get Started Steps page for information on this.

For details of which EOS SDK release versions support which game-client platform release versions, see the Platforms Supported documentation.

Release Versioning

EOS SDK versioning guarantees backwards compatibility with games using an older version of the EOS SDK, even with API changes.

Every interface, function, and data type has a version associated with it. When a new EOS SDK version is released, the new EOS SDK recognizes and supports games using older versions of the EOS SDK and reacts appropriately to data and function calls coming from them. This reaction might include calling the original version of the function internally, migrating the original input structures to newer forms, or setting up defaults.

If your game uses an older version of the EOS SDK, you do not need to update it to the latest version to continue to use Epic Online Services.
