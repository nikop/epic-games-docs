Crossplay Introduction

An introduction to crossplay and the EOS SDK services for it.

10 mins to read

Overview

The Epic Online Services SDK (EOS SDK) provides services to launch, operate, and scale your game across multiple platforms and stores. All the services are built to work across multiple platforms and the EOS SDK unifies player's social information to provide a consistent user experience. See the Epic Online Services (EOS) Overview documentation for more information about EOS. Below is an overview of the services that the EOS SDK provides for crossplay. For more information on the required integration steps to use crossplay in your game, see the Crossplay Technical Overview documentation.

EOS provides four subsets of services for crossplay:

  • Cross store support
  • Epic Online Services (crossplay with the Social Overlay and Epic Games account)
  • Support for Native APIs
  • Platform agnostic multiplayer services

  • The EOS SDK provides cross store support that allows you to unify player's social information so that players can reach their friends across multiple platforms and storefronts.

  • Epic Online Services enable you to use crossplay on multiple platforms with the Social Overlay and Epic Games accounts. You can allow players to invite Epic friends to join multiplayer games from the Social Overlay. The Social Overlay handles all social interactions between your players, for example, sending invites to join a lobby or joining a session via presence. This allows you to enable crossplay with simplified code integration of EOS while still providing programmatic access where needed.

    (See the Epic Account Services and the Social Overlay section below for more information.)

  • Support for Native APIs that enable native invites to reach native friends on multiple platforms. If you use the Social Overlay to allow players to invite their friends, for many platforms you need to also send a native invite which then allows players to view them on the dashboard of that platform.

    (See the Support for Native APIs section below for more information.)

  • Use platform agnostic multiplayer services that do not include social systems but are needed for your multiplayer game, for example, Epic Online Services achievements. You can use these services with your own social system or combine them with the Social Overlay and Epic Games accounts to handle the social interaction between players.

    (See Platform Agnostic Multiplayer Services section below for more information.)

Cross Store Support

Follow these steps to get started with crossplay and implement cross-store support:

  1. Choose Your Authentication Method

    We recommend that you choose how your players can authenticate and sign in to your game first before continuing to develop EOS Game Services. You can allow players to connect with your multiplayer services by using the Connect Interface to allow players to sign into your game from a supported identity provider. You can also allow players to sign in with their Epic Games account directly into your game.

    • Epic Online Services

      Use the Connect Interface for game features like Lobbies, Voice, Achievements, and Leaderboards.

      For more information on how to allow players to sign in from a supported identity provider, see the Connect Interface documentation.

    • Epic Account Services

      Use the Auth Interface for a social graph, presence, Epic friends, and user information.

      For more information on how to allow players to sign in with their Epic Games account, see the Auth Interface documentation.

    Note: When you choose a sign in method, you must continue to use that method throughout the following steps.

  2. EOS Game Services

    We recommend that you build your game and integrate EOS Game Services before progressing with the development of the Social Overlay. The social overlay will be able to interface with this in later steps. You can choose from a number of services, including:

    • Achievements
    • Anti-Cheat
    • Leaderboards
    • Lobbies
    • Metrics
    • P2P
    • Sessions
    • Voice
    • and more (see EOS Game Services for more information).

    Note: EOS Game Services are platform agnostic by default. For more information on the platforms that EOS Game Services support, see the Platforms Supported documentation.

  3. Social Overlay

    You can integrate the Social Overlay into your game across multiple platforms. This allows you to use simple game code integration to enable players to use their Epic Games accounts to invite their Epic friends to join lobbies and sessions as well as their party. A player can also join a lobby or session via presence.

    (See the Epic Account Services and the Social Overlay section below for more information.)

    Note: You must use Epic Account Services in your game to have access to the Social Overlay. See the Epic Account Services documentation for more information.

  4. Integrate with Native APIs

    You can set up the native platform to allow players to send, receive and accept native invites for the lobby and/or session you implemented in Step 1, or use custom invites to integrate your own invite system. You need to work in combination with native platform APIs and the EOS Integrated Platform API to send native invites or provide native presence. You also need to do this to reach native friends and show them on the native dashboard.

    (See Platform Agnostic Multiplayer Services section below for more information.)

Epic Account Services and the Social Overlay

You can use Epic Account Services to allow players to sign directly into your game with an Epic Games account and open the Social Overlay in your game to invite their friends or join a game via presence or request to join it.

Note: When you integrate the EOS SDK with your game and with the following services, you do not need to do any additional game code to manage social interactions between players because the EOS SDK and the Social Overlay do this for you. However, you do need to mirror all information for the native platforms.

  • Authentication

    Use the Auth Interface to allow players to sign into their Epic Games account directly from your game. The user first interacts with the Account Portal to sign in, authenticate and get an access token, if they haven't already signed in or linked their account. After the user has authenticated, the Social Overlay leverages their Epic Games account and its social graph to send and receive game invites, join a game via presence, and request to join a lobby or session.

    For more information, see the Auth Interface documentation.

  • Friends

    Use the Friends Interface to retrieve a list of friends for a player that is signed in. The Friends Interface creates a local cache that you can use with other interface functions, for example, retrieve account IDs and metadata. You can also subscribe to presence updates, described below.

    Note: The Social Overlay manages a list of Epic friends for a player with an Epic Games account. This list includes platform friends. See the Social Overlay and Friends section of the Crossplay Technical Overview documentation for more information. For more information on the Friends tab of the Social Overlay, see the Social Overlay Tabs: Friends documentation.

    For more information, see the Friends Interface documentation.

  • Presence

    Use the Presence Interface to query the presence of players online and share more detailed information about the player. If you integrate the Social Overlay with your game then players can use their Epic Games account to join a lobby or session via presence.

    For more information, see the following documentation:

    • Presence Interface
    • Social Overlay for Crossplay

  • Social Overlay

    Integrate the Social Overlay with your game to allow players to:

    • Manage their friends:
      • See a list of their friends.
      • Send invitations to make new friends.
      • Block nuisance players.
    • See a friend's profile for details about them (for example, what game they are playing).
      Note: The Social Overlay integrates with the native platform to provide profile information.
    • Send invitations to friends to join their game lobby, session, or your custom solution.
    • See a list of achievements in your game and their progress to complete each one.

    For more information on:

    • How to include the overlay technology during EOS SDK initialization for rendering and input considerations, see SDK Integration: SDK Initialization
    • For full information on how to use the Social Overlay with your game, see the SDK Integration: Social Overlay documentation.
    • What a player can do from the Social Overlay, see the Social Overlay documentation.
    • How to allow players to use the Social Overlay with lobbies or sessions, see the Social Overlay for Crossplay documentation.

      Note: If you allow a player to invite their Epic friend to your lobby or session from the Social Overlay and they are also friends on a platform you integrated into your game, then you must comply with native platform requirements including sending a native invite.

Support for Native APIs

If you want to release your game with EOS multiplayer services like lobbies or sessions, the Social Overlay and the native platform social system must work together to allow players to invite other players, request to join a game, or join via presence. To do this you need to work in combination with native platform APIs and the EOS Integrated Platform API to send native invites or provide native presence. You also need to do this to reach native friends and show them on the native dashboard.

  • Initialize the EOS SDK

    When you develop a game with the EOS SDK on multiple platforms, you need to initialize the EOS SDK. The order that you do so is determined by the platform on which you develop your game, either before or after you initialize the graphics layer of your game.

    For more information on how to initialize the overlay for each platform, see SDK Initialization: SDK Integration.

    For more information on how to initialize the EOS SDK for each platform, see:

    • Social Overlay for Crossplay

  • Register Callbacks for Game Services

    When you develop a game with the EOS SDK on multiple platforms and you want to allow players to use the Social Overlay to join a lobby or session, then you need to register a number of callbacks so that the EOS service can notify you.

    For more information on how to register the callbacks, see:

    • Social Overlay for Crossplay

  • Register Callbacks for Integrated Platforms

    When you develop a game with the EOS SDK on multiple platforms you need to register a number of callbacks so that the EOS service can notify you when players with accounts on these platforms join a lobby or session or want to use your custom invitation system.

    For more information on how to register the callbacks, see:

    • Social Overlay for Crossplay

  • Set Flags and Permissions for a Lobby or Session

    When you create a lobby or session, you need to set a number of flags and permissions to allow players to join or request to join them. You can also choose to not allow players on certain platforms to join through crossplay restrictions.

    For more information on how to set the flags, see:

    • Social Overlay for Crossplay
    • Crossplay restrictions

  • Allow Friends to Join from an Integrated Platform

    To allow friends on a native platform to send invitations on that platform and accept them, you need to make calls for some actions taken by the players:

    • When a player invites another player to a lobby or session or sends a custom invite.
    • When a player accepts the invitation.

    You also need to manage the joinability state of the session across platforms.

    • The game needs to mark the native session as "full" or "private" to emulate the fullness and block friends from joining the game during this period.

    For more information on the calls you need to make, see:

    • Social Overlay for Crossplay

  • Shut Down Your Game

    When you shut down your game, it is good practice to remove all callbacks you use to handle notifications for players with Epic Games accounts and players with accounts on platforms that you have integrated with your game (for example Steam accounts).

    The game should also make sure to tear down lobbies and sessions to prevent stale information with backend services.

    For more information on the calls you need to remove, see:

    • Social Overlay for Crossplay

Platform Agnostic Multiplayer Services

Epic Online Services are platform agnostic by default, which means you can integrate the services with your game on the platforms supported by EOS, and use each service across those platforms. For example, if you integrate EOS lobbies or sessions then the matchmaking pools are platform agnostic by default. This means that you can choose to use a service and then combine that service with Epic Account services, the platform's native social systems, or your own social systems.

The following are examples of services provided by the EOS SDK that are platform agnostic and do not necessarily need social interaction:

  • Authentication

    For more information, see the Authentication Interface documentation.

  • Epic Online Services achievements

    For more information, see the Achievements Interface documentation.

  • Lobbies

    For more information, see the Lobby Interface documentation.

  • Sessions

    For more information, see the Sessions Interface documentation.

  • P2P

    For more information, see the Nat P2P Interface documentation.

  • Easy Anti-Cheat

    For more information, see the Anti-Cheat Interfaces documentation.

    Note: The Easy Anti-Cheat only supports Windows, macOS, ChromeOS, and Linux 64-bit operating systems.

  • Player Reports

    For more information, see the Reports Interface documentation.

The following services are also platform agnostic and support social interaction:

  • Voice Chat

    For more information, see the Voice Interface documentation.

  • Social Overlay

    For more information see the Social Overlay Overview documentation.
