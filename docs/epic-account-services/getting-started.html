Get Started

How to set up EOS Epic Account Services with your game.

10 mins to read

EOS Epic Account Services is part of Epic Online Services (EOS). EOS consists of two sets of resources: EOS Epic Game Services, and EOS Epic Account Services. See the following documentation to find out more: EOS Introduction and EOS Get Started Steps.

EOS Epic Account Services are a set of resources which are specific to using Epic Games accounts in your game (or other software product).

If you use EOS Game Services in your game, you do not require EOS Epic Account Services. Players can connect to your game using either an identity from a supported identity provider (such as a Steam account) or an Epic Games account.

Before You Start

Set up your Epic games account in the Developer Portal
Before you can begin setting up an EOS Epic Account Services account and registering your game to use Epic Account Services, you must have set up an Epic Games account and an organization in the Developer Portal.

If you have not done this already, follow step 1 in the following documentation: EOS Get Started Steps.

Get Started

Admin Process
Once you have set up an Epic Games account and an organization in the Developer Portal, you can start the Epic Account Services admin process. This page describes how to do this step-by-step.

Integrate Epic Account Services with your Game
Once you have completed the admin process, you can integrate Epic Account Services into your game using the EOS SDK.

For guidance on accessing and working with the EOS SDK, follow step 2 onwards in the following documentation: EOS Get Started Steps.

When you are familiar with the EOS SDK, you can integrate Epic Account Services with your game. This page also describes how to authenticate players using Epic Account Services.

Tip

  • To let players connect to your game with their Epic Games accounts, use the EOS Epic Account Services: Auth Interface.
  • To let players connect to your game with identities provided by online stores and gaming services such as Steam, use the EOS Game Services: Connect Interface.

For more information, see the following documentation:

  • For details of the identity providers supported by EOS, see Identity Provider Management.
  • EOS Games Services: Connect Interface.
  • EOS Epic Account Services: Auth Interface.
  • EOS SDK API Reference: EOS_EExternal Credential Type.

Admin Process

1. Add your game to your organization

Add the game you want to use EOS Epic Account Services to your organization in the Developer Portal.
To do this:

  1. In the Developer Portal, click + Create Product.
  2. Enter the name of your game (known as a product or application in the Developer Portal) and click Create.

2. Access EOS Epic Account Services in the Developer Portal

You can find Epic Account Services from the Developer Portal along the navigation panel or the home page for a product.

To proceed to the Epic Account Services dashboard, you must review and accept a Service Addendum for Epic Account Services. You can always review all accepted agreements under the Licenses tab of the Organization section in the Developer Portal.

You should also review the Epic Games Privacy Policy to understand how Epic Games collects, uses, and shares user data.

Depending on your organization type, you may only need to accept each license agreement once (as a Company entity). However, Individual entities require each member with access to the EAS dashboard to accept the agreement.

With EOS Game Services, you license Epic Games as a service provider to process the data of your users on your behalf. But with EOS Epic Account Services, Epic Games manages the player data and authentication flows to expose this data and functionality on your players' behalf, with their consent. From your end, you agree to use the data for permitted uses only, as defined in the Service Addendum for Epic Account Services.

3. Set Up Your Game with EOS Epic Account Services

From the Epic Account Services dashboard, you can register your game for Epic Account Services. (Note that your game is usually known as a product in the Developer Portal but your game is known as an application when you are setting it up for Epic Account Services.)
The Developer Portal will show a placeholder for your game as an "application" using Epic Account Services, but none of the configuration details.

Click to enlarge image.

Your game's configuration with Epic Account Services consists of three major sections:

  1. Brand Settings: Name, logo, privacy policy URL, and support URL.
  2. Permissions: The access permissions that the application asks users for.
  3. Linked Clients: Association of the application with client credentials used by the SDK during the authentication flow.

Click Configure to proceed with each of these.

Brand Settings

The first section for configuring your game is Brand Settings.

Click to enlarge image.

On the left you can see a preview of the consent dialog that will be shown to Epic Games users requesting their permission to share information with your game. The preview is available in a mobile and a web version.

Initially, a red warning banner will appear at the top of the preview for the consent dialog. This warning indicates that the game's brand settings have not been reviewed against potential toxicity or an attempt to spoof the brand. Additionally, the consent dialog for an unverified game is prefaced with another warning dialog. The purpose of the double warning is to fully ensure that the epic Games user understands all the implications of proceeding with an unverified game.

All games are created unverified, and with audience restrictions in place. Only members of the same development organization will be able to use their Epic Games accounts to authenticate against this unverified application. All other Epic Games users will see an audience restriction error message. The purpose of these audience restrictions is to protect Epic Games users while allowing developers to iterate on integration while they prepare their game for brand review.

To remove these restrictions, the details in the fields on the right must be filled out and submitted for Brand Review. These include:

  • Application Name: A friendly name for your game that will display to users. This is set to the product name by default.
  • Privacy Policy URL: A URL where users can find your game's privacy policy.
  • Application Logo: A 128 x 128 opaque PNG or JPG that will represent your game on the consent dialog.

As you update these fields, the preview will immediately update to reflect your changes.

Click to enlarge image.

Once the game is created and brand settings are saved, go to the next tab.

Permissions

The Permissions tab is where you define the level of access the game will request from the Epic Games users.

Click to enlarge image.

Epic Account Services ships with support for three basic permissions:

  • Basic Profile: Allows read access to user display name, language preferences and linked account display names.
  • Online Presence: Allows the application to set the online presence of the current player and receive online presence updates from their friends.
  • Friends: Allows read access to the friends list for the current player's account.

Basic Profile is always required for Epic Account Services and can not be deactivated as you cannot get access to any user data without it. Online Presence and Friends may be enabled or disabled per the needs of your application. Toggle on the permissions you want to enable for your application, then click Save to save your changes. The consent dialog preview will update to reflect the permissions you add or remove.

Click to enlarge image.

Linked Clients

Finally, the Linked Clients tab enables you to configure a list of clients associated with this game. A client is any piece of software or website that can access the games's data within a service provided by Epic Account Services.
All clients associated with a game are listed in its Product Settings, and all of them share the Epic Games user's consent once it is given.
Note: "Application Clients" and "Linked Clients" mean the same thing.)

Click to enlarge image.

If you already have clients set up for your game, you can select them from the Select Clients dropdown. Otherwise, click Create New Client to open your Product Settings page and configure a new client. For more information on how to add new clients, see the Client Credentials page.

Once you have added a client to this list, click Save to save your changes.

Click to enlarge image.

At this point you have completed the admin process. From here you are ready to integrate EOS Epic Account Services into your game and use it to access Epic Games user data. Following a review of the brand settings you submitted, audience restrictions will be lifted, and the game can be made visible to Epic Games users outside of your organization.

However, you must use a login type appropriate for your target platform (see the Auth Interface), and you must remember to include the scopes you want the Epic Games user to consent to in the login call that you denoted during your brand review (their profile, presence , and friends list).

Authenticate Epic Games Users on a Web Site

To authenticate Epic Games users on websites and web applications, we support the authorization code flow for OpenID Connect Core 1.0. But before you begin, you must configure a brand review and client credentials.

User Authorization

To initiate the authentication flow, your application must redirect the Epic Games user to the authorization page where they will login to their Epic Games account.

Redirect your Epic Games users to this authorization URL:

To request additional permissions, modify the scope parameter in the authorization request with a space-delimited list of the required permissions. For example, scope=basic_profile friends_list.

You then need to configure the redirect URL for the Client in your Developer Portal.

When a user logs in, they are asked to approve the requested permissions. After accepting, they are redirected back to your application with a code parameter. This code is used when requesting an access token.

EOS also supports an optional state parameter that you can use to maintain state between the request and the callback. This can prevent cross-site request forgery attacks.

Requesting an Access Token

To request an access token, the client makes a request to the token endpoint, including the client credentials and authorization code. The Epic Games token endpoint is https://api.epicgames.dev/epic/oauth/v1/token.

The client credentials pass in the Authorization header using basic authorization.

In the request, you also need to specify the authorization_code grant type and include the code and redirect URL from the authorization flow.

The response contains the following fields:

Response Description
access_token The access token, which may optionally have a prefix (for example: eg1~token). This value should be passed as-is in the Authorization header using the Bearer type on any requests to Epic services.
expires_in The number of seconds until the token expires.
expires_at The expiration date in ISO 8601 format.
account_id The Epic Games account ID for the user that the token was generated for.
client_id The Client ID which was used to generate this token.
application_id The Application ID which the Client is associated with.
token_type The type of token generated, value will always be bearer.
refresh_token The refresh token will optionally be returned depending on the client configuration. This refresh token can be used to extend a session before or after the access token has expired.
refresh_expires The number of seconds until the refresh token expires.
refresh_expires_at The refresh token expiration date in ISO 8601 format.
