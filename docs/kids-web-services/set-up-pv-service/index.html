Set Up the Parent Verification Service

How to set up the KWS Parent Verification service for your product, first in a Test environment and then in your Production environment.

Under 1 min to read

Each product within KWS comes with a Test environment and a Production environment. Each environment comes with its own configuration and integration details. This enables you to test the Parent Verification (PV) service thoroughly on your product in the Test environment before you push the service live.

The roadmap for setting up the PV service is:

  1. Consider how the PV service will fit into your product's flow. See the example use case for guidance.
  2. Get started with your PV service configuration.
  3. Configure the PV service for your product. This consists of three steps:
    1. Choose which verification methods will be available to parents.
    2. Provide a redirect URL where KWS will direct parents after verification (optional).
    3. Provide a URL to receive a success notification webhook from KWS, and generate a signature to validate the webhook call.
  4. Integrate with the KWS API.
  5. Test the PV service using dummy email addresses and verification credentials.
  6. When you have tested the service to your satisfaction, set up your Production environment in the same way as the Test environment.
  7. After KWS reviews and approves your Production environment, you're ready to go live.
