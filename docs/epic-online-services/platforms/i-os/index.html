EOS SDK for iOS

How to implement EOS SDK in an iOS project using Xcode.

3 mins to read

To use Epic Online Services (EOS) SDK in your project, you must use the EOS SDK framework for iOS (the framework). These instructions assume that you have an existing iOS project in Xcode.

Before implementing the SDK with iOS, be sure to review the general guidelines and references for platform implementation.

Objective-C and Swift iOS Projects

The EOS SDK for iOS is written in C. If your game's code is written in Objective-C (ObjC) or Swift, we recommend writing an ObjC wrapper around the EOS SDK, as the EOS SDK is not a Swift module. The Login Sample, included in the EOS SDK download for iOS, uses an ObjC wrapper. See the Login Sample's /Login/EOS/EOSWrapper.h file for an example implementation. See Using the iOS Samples documentation for more information on using the Login Sample app.

Download the EOS SDK Framework for iOS

Follow these steps to download the EOS SDK framework for iOS (the framework):

  1. Sign in to the Developer Portal at https://dev.epicgames.com.
  2. Click DOWNLOAD SDK.
  3. From the SDK type drop-down menu, select the SDK for iOS option.
  4. From the Version drop-down menu, select the latest version, and click DOWNLOAD.

Add the EOS SDK Framework to Your Project

After you have downloaded the EOS SDK framework, you must add it to your project. The example below references a sample game project called EOSGame. If you know how to link and embed an iOS framework, you can skip to the Framework Dependencies section below.

Follow these steps to add the EOS SDK framework to your project:

  1. Go to the ./SDK/Bin/IOS/EOSSDK.framework folder inside the EOS SDK zip you downloaded.

  2. Copy the EOSSDK.framework folder into your project's root folder.

  3. Open your project in Xcode.

  4. In Xcode, select your project in the left column to open its properties.

  5. Select your target in the middle column, under the TARGETS section.

  6. Select the General tab and scroll to the Frameworks, Libraries and Embedded Content section at the end.

  7. Drag the EOSSDK.framework folder from your project folder in Finder and drop it into the Frameworks, Libraries and Embedded Content section in Xcode.

  8. Click the Build Phases tab, and configure these settings for EOSSDK.framework:

    • From the Link Binary With Libraries > Status drop-down menu, select Required.
    • Check Embed Frameworks > Code Sign On Copy.

Add Framework Dependencies

The EOS SDK requires the AuthenticationServices.framework to support the following features for iOS:

  • iOS 12+: Login flows.
  • iOS 13+: Sign in with Apple login.

Note: If your game supports iOS 11, you must also include SafariServices.framework for the login flows. Follow the same steps below to add SafariServices.framework to your project. Note that this API is deprecated and may be removed by Apple in the future. It is only supported for limited backwards compatibility.

Follow these steps to add the AuthenticationServices dependency to your project:

  1. Click the + (the plus icon) at the end of the Frameworks, Libraries and Embedded Content section.

  2. Select AuthenticationServices.framework from the list, and click Add to add it to your project.

  3. You can now see the dependency listed in the Frameworks, Libraries, and Embedded Content section, along with the EOSSDK.framework. Change the Embed option for the AuthenticationServices framework to Do Not Embed.

Next Steps

Build and run the app on your device to ensure that it builds properly. Simulator support in Xcode is currently not available to test the EOS SDK. To run the EOS SDK framework in your project, you must have a physical, connected, device with iOS 11 or higher.

Note: You must initialize the EOS SDK before you can use the interfaces. See the Platform Interface documentation for more information on initializing the EOS SDK.
