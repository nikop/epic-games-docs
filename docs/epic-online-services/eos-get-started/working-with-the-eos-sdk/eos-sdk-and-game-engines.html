EOS SDK and Game Engines

An overview of how to use Epic Online Services (EOS) in your game engine.

5 mins to read

The Epic Online Services (EOS) SDK is a game engine-agnostic tool that gives you access to several cross-platform online services that you can use in your game. The game engine you choose determines the available options you have to integrate the EOS SDK into your game. You can even use a combination of integration options, if you prefer. You are not required to use the plugins described in the sections below.

Any Game Engine

You can use the EOS SDK in your game, regardless of the engine you choose to develop your game in (even if you've written the engine yourself).

To get started, download the EOS SDK for your programming language and set up your game (or other software application) as a product on the Developer Portal. See the Get Started Steps documentation for more information.

Unreal Engine

If you develop your game in Unreal Engine, you can use the following options to integrate the EOS SDK into your game:

  • Manual configuration
  • The Online Subsystem (OSS) plugins
  • The Online Services plugins

Which Set of Plugins to Use

  • If you use an Unreal Engine version lower than 5.1, we recommend you use the Online Subsystem (OSS) plugins.
  • If you use Unreal Engine 5.1 or higher, the Online Services plugins provide several improvements over the OSS plugins (see the notes below on Online Services plugins for more information).

Both the Online Subsystem (OSS) plugins and Online Services plugins provide several interfaces that you can implement in your game's code to let your players access online services in your game. The services your game can access include lobbies, sessions, leaderboards, and stats, among others. See the Unreal Engine documentation on Using Online Subsystem or Online Services to help decide which plugin is right for your game.

Note: You can only access console-specific OSS implementations if you have the appropriate permissions. Contact Epic Games support for assistance. See the Support documentation for a list of support resources.

For information on how to upgrade the EOS SDK for Unreal Engine, see Unreal Engine's Upgrading the EOS SDK documentation.

Manual Configuration

See the Get Started Steps documentation for more information. After you've completed these steps, you can use the EOS SDK interfaces in your game's code.

Note: You must initialize the EOS SDK in a project plugin (See Unreal Engine's documentation: Plugins). If you add the code to a game object, the Social Overlay might not load.

The Online Subsystem (OSS) plugins (UE 4.27 and higher)

Unreal Engine provides the following Online Subsystem (OSS) plugins that you can use to integrate the EOS SDK into your game:

  • The EOS Online Subsystem plugin (UE 4.27 and higher):
    • Configure your game's EOS settings in the Unreal Editor (no code required). See Unreal Engine's Online Subsystem EOS Plugin documentation for more information.
    • Use the Online Subsystem interfaces to include EOS services in your game. See Unreal Engine's Online Subsystem Interfaces documentation for more information.
  • The EOS Plus Online Subsystem plugin (UE 4.27 and higher):
    • Note: You don't need this plugin if you only ship your game on the Epic Games Store.
    • Use this plugin to combine the EOS Online Subsystem with another platform OSS (for example, Xbox Live or Steam).
    • Configure your game's EOS Plus settings in the Unreal Editor. See Unreal Engine's Online Subsystem EOS Plus Settings documentation for more information.

Note: The EOS Online Subsystem plugin exposes EOS SDK interfaces through the Online Subsystem interfaces. However, the list of interfaces listed below are not currently included in the EOS Online Subsystem. Therefore, you must use the following EOS SDK interfaces to use the corresponding services in your game:

  • Anti-Cheat: See the Anti-Cheat Interfaces documentation for more information.
  • Custom Invites: See the Custom Invites Interface documentation for more information.
  • Reports Interface: See the Reports Interface documentation for more information.
  • Sanctions Interface: See the Sanctions Interface documentation for more information.

The Online Services plugins (UE 5.1 and higher)

Note:

  • We have not tested these plugins in production. They are for testing purposes only.
  • We recommend that you use these plugins in either of the following circumstances:
    • You use your own (non EOS) services.
    • You plan to incorporate Unreal Engine upgrades beyond UE 5.1 into your game before you ship.

The Online Services plugins split EOS functionality into two plugins based on which services you want to use:

  • For EOS Game Services: The Online Services EOSGS plugin (UE 5.1 and higher).
    See the EOS Game Services documentation for more information.
  • For EOS Epic Account Services: The Online Services EOS plugin (UE 5.1 and higher).
    See the Epic Account Services documentation for more information.

Use the Online Services interfaces to include EOS services in your game.
For more information:

  • See Unreal Engine's Online Services Interfaces documentation.
  • See Unreal Engine's Online Services documentation.

Unity

If you develop your game in Unity, you can use the following options to integrate the EOS SDK into your game.

Manual configuration

See the Get Started Steps documentation for more information. After you've completed these steps, you can use the EOS SDK interfaces in your game's code.

The EOS Plugin for Unity (third-party, developed by PlayEveryWare)

Configure your game's EOS settings in the Unity editor (no code required). See PlayEveryWare's documentation (eospluginforunity.playeveryware.com) for more information on how to configure the EOS Plugin for Unity.

To integrate the EOS SDK into your game's code, use one of the following options:

  • The interface wrappers demoed in the EOS for Unity plugin's EOS samples: Note that you must install the EOS samples separately with the UPM (Unity Plugin Manager). See the Samples section of PlayEveryWare's documentation for more information:(eospluginforunity.playeveryware.com/~samples).
  • Standard EOS SDK implementation: See the Interface documentation for more information on how to use each interface and the Samples documentation for examples of how to use each interface. See the EOS SDK in C# documentation for general guidance on how to work with EOS in C#.

Note: If you use the wrappers demoed in the Unity EOS samples, the implementation is not exactly the same as what we describe in our EOS SDK interface documentation. If you use Unity's wrappers for the EOS SDK, you must follow their naming conventions to use EOS services in your game.
