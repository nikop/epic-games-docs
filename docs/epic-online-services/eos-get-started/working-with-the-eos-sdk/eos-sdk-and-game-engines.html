EOS SDK and Game Engines

An overview of how to use Epic Online Services (EOS) in your game engine.

3 mins to read

The Epic Online Services (EOS) SDK is a game engine-agnostic tool that gives you access to several cross-platform online services that you can use in your game. The game engine you choose determines the available options you have to integrate the EOS SDK into your game. You can even use a combination of integration options, if you prefer. You are not required to use the plugins described in the sections below.

Any Game Engine

You can use the EOS SDK in your game, regardless of the engine you choose to develop your game in (even if you've written the engine yourself). To get started, download the EOS SDK for your programming language and set up your product on the Developer Portal. See the Get Started Steps documentation for more information.

Unreal Engine

If you develop your game in Unreal Engine, you can use the following options to integrate the EOS SDK into your game.

  • Manual configuration: See the Get Started Steps documentation for more information. After you've completed these steps, you can use the EOS SDK interfaces in your game's code.
    • Note: You must initialize the EOS SDK in a project plugin (See Unreal Engine's documentation: Plugins). If you add the code to a game object, the Social Overlay might not load.
  • The EOS Online Subsystem plugin (UE 4.27 and higher):
    • Configure your game's EOS settings in the Unreal Editor (no code required). See Unreal Engine's Online Subsystem EOS Plugin documentation for more information.
    • Use the Online Subsystem interfaces to include EOS services in your game. See Unreal Engine's Online Subsystem Interfaces documentation for more information.
  • The EOS Online Services plugins (UE 5.1 and higher):
    • EOS functionality is split into two plugins based on what you want to use:
      • Online Services EOS: this provides access to Epic Account Services. See the Epic Account Services documentation for more information.
      • Online Services EOSGS: this provides access to EOS Game Services. See the EOS Game Services documentation for more information.
    • Use the Online Services interfaces to include EOS services in your game. See Unreal Engine's Online Services Interfaces documentation for more information.
    • See Unreal Engine's Online Services documentation for more information.

If you want to use either the Online Subsystem EOS plugin or the Online Services EOS plugins, see the Using Online Subsystem or Online Services documentation to help decide which plugin is right for your project.

Note: The EOS Online Subsystem plugin exposes EOS SDK interfaces with the Online Subsystem interfaces. However, the list of interfaces below aren't currently included in the EOS Online Subsystem. Therefore, you have to use the following EOS SDK interfaces to use the corresponding services in your game:

  • Anti-Cheat: See the Anti-Cheat Interfaces documentation for more information.
  • Custom Invites: See the Custom Invites Interface documentation for more information.
  • Reports Interface: See the Reports Interface documentation for more information.
  • Sanctions Interface: See the Sanctions Interface documentation for more information.

For information on how to upgrade the EOS SDK for Unreal Engine, see Unreal Engine's Upgrading the EOS SDK documentation.

Unity

If you develop your game in Unity, you can use the following options to integrate the EOS SDK into your game:

  • Manual configuration: See the Get Started Steps documentation for more information. After you've completed these steps, you can use the EOS SDK interfaces in your game's code.
  • The EOS Plugin for Unity (third-party, developed by PlayEveryWare):
    • Configure your game's EOS settings in the Unity editor (no code required). See PlayEveryWare's documentation for more information on how to configure the EOS Plugin for Unity.
    • To implement the interfaces in your code, you have two options:
      • Use the interface wrappers demoed in the EOS for Unity plugin's EOS samples. Note that you must install the EOS samples separately with the UPM (Unity Plugin Manager). See the Samples section of PlayEveryWare's documentation for more information.
      • Standard EOS SDK implementation: See the Interface documentation for more information on how to use each interface and the Samples documentation for examples of how to use each interface. See the EOS SDK in C# documentation for general guidance on how to work with EOS in C#.

Note: If you use the wrappers demoed in the Unity EOS samples, the implementation is not exactly the same as what we describe in our EOS SDK interface documentation. If you use Unity's wrappers for the EOS SDK, you must follow their naming conventions to use EOS services in your game.
