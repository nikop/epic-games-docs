BuildPatch Tool Release Notes

What's changed in updates to the BuildPatch Tool.

2 mins to read

1.6.0: 2024-february-23

Important Information

  • Development environment system requirements:

    • macOS (Updated)
      • High Sierra 10.13 or higher
    • Other operating systems (No change)
      • Windows (No change)
        • Win 7 64-bit or higher
      • Linux (No change)
        • CentOS 7 64-bit or higher
        • Debian 10 64-bit or higher
        • Fedora 32 64-bit or higher
        • Ubuntu 16.04 64-bit or higher
        • Any Linux distribution that meets Docker`s prerequisites for a Docker Engine install. (See docs.docker.com/~/linux-install for more information.)

  • Argument FileAttributeList

    When you use FileAttributeList to list file paths, you must use double quotation marks around the file path and filename.

    • Example use:
      • "Relative/Path/To/My/file.bin" executable tag:my_tag
      • "Second/Path/To/A/file.txt" tag:txt_files
    • Documentation: Build Patch Tool Instructions 1.6.0: Setting Special File Attributes.

New Features

New Mode

  • Use the command-line argument GetBinaryMetadatato see information about a binary . The information is in human-readable or json format.
  • Use UninstallActionPath and UninstallActionArgs optional arguments for UploadBinary mode to set up an executable file to launch just before product uninstall.

Bug Fixes and Improvements

  • File uploads:
    • Improved performance to ensure the BuildPatch Tool triggers high priority upload requests quicker.
    • Improved error messaging.
    • Fixed an issue which caused some modes to hang during file uploads. This no longer happens.
  • Command-line arguments:
    • UnlabelBinary and LabelBinary modes: Improved error messaging for missing permissions response.
    • ListBinaries mode: Now displays sandbox information.
    • DiffBinaries mode: Improved performance.
    • FileAttributeList and FileIgnoreList files:
      • Added support for regex and wildcards so you don’t need to explicitly list all files in a build.
      • Added additional verification and error reporting when you use these arguments.
  • Reduced log noise:
    • Fixed an issue where retry requests during chunk upload created logs excessively. Verbose logging now shows on the last attempt before retries are exhausted.
  • Set language locale default:
    • The BuildPatch tool’s language locale now defaults to en. This avoids crashes when the BuildPatch tool runs on PCs with unsupported language locales.
  • Performance improvements:
    • Faster SHA1 hash-based file verification implementation.
