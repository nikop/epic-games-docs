EOS_IntegratedPlatform_AddNotifyUserLoginStatusChanged

EOS API reference page for EOS_IntegratedPlatform_AddNotifyUserLoginStatusChanged

2 mins to read

This function is part of the IntegratedPlatform Interface.

Remarks

Register to receive notifications when the login state of Integrated Platform users change. This notification will trigger any time the EOS SDK's internal login state changes for a user, including for manual login state changes (when the EOS_IPMF_ApplicationManagedIdentityLogin flag is set), or automatically detected ones (when not disabled by the EOS_IPMF_ApplicationManagedIdentityLogin flag).

Return Value

A valid notification that can be used to unregister for notifications, or EOS_INVALID_NOTIFICATIONID if input was invalid.

This function enables your callback (the CallbackFunction parameter) to respond to events that happen on the back-end service. Remember to remove your callback function when it is no longer needed.

Callback Function Information

Because this function is asynchronous, it employs a callback of type EOS_IntegratedPlatform_OnUserLoginStatusChangedCallback to report the results of its operation.

Callback Remarks

The callback function for when a local integrated platform user's login status has changed.
