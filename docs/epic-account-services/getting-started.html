Get Started

How to set up EOS Epic Account Services with your game.

13 mins to read

EOS Epic Account Services is a set of resources that is specific to using Epic Games accounts in your game (or other software product).

EOS Epic Account Services is part of Epic Online Services (EOS). EOS consists of two sets of resources: EOS Epic Game Services, and EOS Epic Account Services. See the following documentation to find out more: EOS Introduction and EOS Get Started Steps.

When to Use EOS Epic Account Services

If you use only EOS Game Services in your game, you do not require EOS Epic Account Services. You only need to use EOS Epic Account Services if you want your players to connect to your game using their Epic Games accounts so they can use services such as crossplay, friends, player presence, and the social overlay.

EOS Game Services
When you connect players to your game using EOS Game Services, note the following points:

  • Players can connect to your game using either an identity from a supported identity provider (such as a Steam account, or other supported identity provider) or an Epic Games account.
  • You use the Game Services Connect Interface to connect players through a supported identity provider.

Documentation:

  • Epic Online Services Overview: EOS Game Services
  • Identity providers supported by EOS:
    • Identity Provider Management
    • EOS SDK API Reference: EOS_EExternal Credential Type.
  • EOS Games Services: Connect Interface.

EOS Epic Account Services
When you connect players to your game using EOS Epic Account Services, note the following points:

  • Players can connect to your game using Epic Games accounts.
  • Players can use a set of resources that are specific to Epic Games accounts, such as crossplay, friends, player presence, and the social overlay.
  • You use the Epic Account Services Auth Interface to connect players through Epic Games accounts.

Documentation:

  • Epic Online Services Overview: EOS Epic Account Services.
  • EOS Epic Account Services: Auth Interface.

Before You Start

Sign Up or Sign In to an Epic Games Account in the Developer Portal

Before you can begin setting up EOS Epic Account Services with your game, you must sign in to your Epic Games account in the Developer Portal and set up an organization.

If you don’t have an Epic Games account already, you can sign up through the Developer Portal. This is an Epic Games account and organization for you to manage EOS services with your game. You also have the option to manage your game on the Epic Games Store through the Developer Portal.

How to Sign Up or Sign In

If you have not already signed-up or signed-in on the Developer Portal, follow Step 1 - Set up an Epic Games Account and Organization in the following documentation: EOS Get Started Steps.

Get Started Overview

There are two parts to get started with Epic Account Services: first you must follow an admin process, and then you can integrate Epic Account Services with your game.

Part 1: Admin Process
Once you have set up an Epic Games account and an organization in the Developer Portal, you can start the Epic Account Services admin process. The section below describes how to do this step-by-step.

Part 2: Integrate Epic Account Services with Your Game
Once you have completed the admin process, you do the following actions:

  • Integrate Epic Account Services into your game using the EOS SDK.
    For guidance on accessing and working with the EOS SDK, follow Step 2 - Download the EOS SDK onwards in the following documentation: EOS Get Started Steps.
  • When you are familiar with the EOS SDK, you can integrate Epic Account Services with your game. This page describes how to authenticate players with Epic Games accounts (known as "Epic Games users") so they can use Epic Account Services in your game.

Get Started: Admin Process

1. Add Your Game to Your Organization

If you have not done this already, add the game you want to use EOS Epic Account Services to your organization in the Developer Portal.

Note: Your game is known as your “product” in the Developer Portal.

To do this:

  1. In the Developer Portal (dev.epicgames.com/portal), at the top of the left navigation, select your organization name.
  2. Click + Create Product.
  3. Enter the name of your product (game or other software application), and click Create.

2. Accept the EOS Epic Account Services Legal Agreement

To accept your game’s EOS Epic Account Services legal agreement, follow the steps below:

  1. In the Developer Portal (dev.epicgames.com/portal), in the top left navigation, click your product’s name. This display’s your product’s page.
  2. From the left-hand menu, click Epic Account Services. This displays the Epic Account Services dashboard.
  3. Review and accept a Service Addendum for Epic Account Services. (If you have already accepted it, it doesn't display again.) To accept the agreement, scroll down to the bottom of the left pane, and then click Accept on the right pane.
Agreement Types

There is a key difference between the EOS Game Services agreement and the EOS Epic Account Services agreement.

  • EOS Game Services agreement:
    • You license Epic Games as a service provider to process your player data on your behalf. There is no agreement between your players and Epic Games.
  • Service Addendum for Epic Account Services:
    • Epic Games manages player data on your players' behalf, with their consent. There is an agreement between your players and Epic Games.
    • You agree to use the player data for permitted uses only, as defined in the Service Addendum for Epic Account Services.
How to Review Agreements

You can review the agreements you have already accepted in the Developer Portal from the Organization page.
To do this:

  1. Go to the Developer Portal Organization page. (dev.epicgames.com/portal menu: < /your organization name/ > Organization.)
  2. Click Agreements. This displays the Agreements tab.

You should also review the Epic Games Privacy Policy to understand how Epic Games collects, uses, and shares user data. (See www.epicgames.com/~/privacypolicy.)

Accept the Agreement More Than Once

Depending on your organization, you might need to accept each agreement more than once:

  • Organizations that are company entities only need to accept each agreement once.
  • Organizations that are individual entities require each member of the organization with access to the Epic Account Services dashboard to accept each agreement.

3. Register Your Product with EOS Epic Account Services

You register your product for Epic Account Services in the Epic Account Services page of the Developer Portal.

To access the page:

  1. In the Developer Portal (dev.epicgames.com/portal), at the top of the left navigation, select your organization name. This displays a list of your products in the left navigation.
  2. Click the name of the product you want to use with Epic Account Services. This displays your product page and the service options in the left navigation.
  3. In the left navigation, click Epic Account Services. This displays the Epic Account Services page (shown below for a product called “Battle Breakers”).
  4. Before you register your game, the Epic Account Services page shows a placeholder for your game as an "application" with no Epic Account Service configuration.

Note: Although your game is usually known as a “product” in the Developer Portal, your product’s settings for EOS Epic Account Services are known collectively as your “application”.

The Epic Account Services Page

Your product’s configuration with Epic Account Services consists of three sections, each with its own tab in the page:

  1. Brand Settings: Your company name, logo, privacy policy URL, and support URL.
  2. Permissions: The user consent that your game requests.
  3. Linked Clients: The clients that connect with your game.

To configure these, click Create Application for a new application or Configure for an existing application. This displays a pane with a pane with a tab for each setting.

Brand Settings

The Brand Settings tab in the Developer Portal. |Click image to enlarge.|

See the documentation: Brand Review Process: Brand Settings for detailed guidance on completing these fields and the brand review process.

Permissions

The Permissions tab in the Developer Portal. |Click image to enlarge.|

In the Permissions tab, you define the level of access your game requests from Epic Games users.

Epic Account Services has three levels of permissions:

  • Basic Profile: Allows read access to user display names, language preferences, and linked-account display names.
  • Online Presence: Allows your product to set the online presence of a user and to receive online presence updates from their friends.
  • Friends: Allows read access to the friends list for a user’s account.

Note: Basic Profile is always required for Epic Account Services. You cannot deactivate it because your product cannot access any user data without it.

You can enable or disable Online Presence and Friends, depending on the requirements of your product.

To set the permission level:

  1. Toggle on the permissions you want to enable for your application.
  2. Click Save changes.
  3. The consent dialog preview on the left side of the tab updates to reflect the permissions you add or remove.
Linked Clients

The Linked Clients tab in the Developer Portal |Click image to enlarge.|

In the Linked Clients clients tab, you choose clients to link to your Epic Account Services application from a list of clients you have already associated with your product.

If you have not already associated clients with your product, you can follow a link from this tab to do this.

About Clients
A client is any software program that accesses EOS services for your product. A client can be:

  • User clients:
    • Game clients used by players on their local devices to play your game.
    • Software clients used by software users on their local devices to run your software application.
  • Trusted servers as clients:
    • Your product might use non-EOS services that are complementary to EOS services. These non-EOS services are known as “trusted servers”. Trusted servers require access to your product's user data, so you must add them as a "client".
  • Any other use for a client you choose.

You must associate a client with client policies: client policies are the necessary permissions for a client.

For further information, see the documentation: Developer Portal - Client Credential and Client Policy Management.

About Linked Clients

  • Linked clients are clients you have already associated with your product that you wish to add to your Epic Account Services application.
    You manage clients in the Clients tab of the Product Settings page. (dev.epicgames.com/portal menu: /your organization name/ > Products: /your product name/ > Product Settings > Clients.)** **
  • The linked clients that you add to an application share the same user permissions (consent) once a user gives consent.
  • You can add as many linked clients to an application as you choose but you can only add them one at a time.

How to Link Clients
Note: In this Developer Portal, "Application Clients" and "Linked Clients" mean the same thing.

  1. If you already have clients associated with your product, you can select them from the Select Clients dropdown.
  2. If not, click add Client to open your product’s Product Settings page and configure a new client.

For details on how to add new clients, see the Client Credential and Client Policy Management page.

Once you have added a linked client to the application, click Save to save your changes.

4. Admin Completed

The Epic Account Services dashboard in the Developer Portal showing configuration complete, with brand review in progress. |Click image to enlarge.|

Once you have completed the fields in the three admin tabs, you have completed the admin process. You are ready to integrate Epic Account Services into your game.

When your brand settings have passed the brand review process, Epic Games removes audience restrictions, and you can make your game visible to Epic Games users outside of your organization.

Integrate Epic Account Services with your Game

Once you have completed the admin process described above, there is no additional setup to use Epic Account Services. You use the EOS SDK with Epic Account Services in the same way that you use the EOS SDK with Game Services. However, with Epic Account Services, you must set up authentication for players using Epic Games accounts (these players are known as "Epic Games users").

If you are authenticating through the EOS SDK, you must use the Auth Interface. You cannot use the Connect Interface: the Connect Interface is only for Game Services.

The following sections give guidance on authenticating Epic Games users, so they can use Epic Account Services in your game.

Authenticate With the EOS SDK

You must do the following:

  • Use a login type appropriate for your target platform. (See documentation: Auth Interface.)
  • Remember to include the scopes you want the Epic Games user to consent to in the login call that you denoted during your brand review (their profile, presence , and friends list). (See documentation: Brand Review.)

For example, to login via the account portal on a PC, your login call may appear as follows:

EOS_Auth_Credentials Credentials = { 0 };Credentials.ApiVersion = EOS_AUTH_CREDENTIALS_API_LATEST;Credentials.Type = EOS_LCT_AccountPortal; EOS_Auth_LoginOptions LoginOptions = { 0 };LoginOptions.ApiVersion = EOS_AUTH_LOGIN_API_LATEST;LoginOptions.ScopeFlags = EOS_AS_BasicProfile | EOS_AS_FriendsList | EOS_AS_Presence;LoginOptions.Credentials = &Credentials; EOS_Auth_Login(AuthHandle, &LoginOptions, NULL, OnLoginCallback);

This opens a default system web browser that prompts the Epic Games account user to login and review the game's access requests.

Authenticate on a Web Site

To authenticate Epic Games users on websites and web applications, EOS supports the authorization code flow for OpenID Connect Core 1.0 (see openid.net).

Note: Before you begin, you must go through brand review, set up user permissions, and set up linked clients, as described above in the Admin Process section of this page.)

User Authorization

To initiate the authentication flow, your game must redirect the Epic Games user to the authorization page where they log in to their Epic Games account.

Redirect your Epic Games users to this authorization URL:

https://www.epicgames.com/id/authorize?client_id={client_id}&response_type=code&scope=basic_profile&redirect_uri=https://www.example.com&state=rfGWJux2WL86Zxr6nKApCAnDo8KexEUE

To request additional permissions, modify the scope parameter in the authorization request with a space-delimited list of the required permissions. For example, scope=basic_profile friends_list.

You then need to configure the redirect URL for the client in the Developer Portal.

When a user logs in, they are asked to approve the requested permissions. After accepting, they are redirected back to your application with a code parameter. This code is used when requesting an access token.

EOS also supports an optional state parameter that you can use to maintain state between the request and the callback. This can prevent cross-site request forgery attacks.

The following is an example redirect URL after a user is authenticated:

https://www.example.com/?code=cfd1de1a8d224203b0445fe977838d81&state=rfGWJux2WL86Zxr6nKApCAnDo8KexEUE
Request an Access Token

To request an access token, the client makes a request to the token endpoint, including the client credentials and authorization code. The Epic Games token endpoint is https://api.epicgames.dev/epic/oauth/v1/token.

The client credentials pass in the Authorization header using basic authorization. (Documentation: Auth Interface.)

In the request, you also need to specify the authorization_code grant type and include the code and redirect URL from the user authorization flow, described above. (See the section in this page: User Authorization.)

The following snippet shows a sample request (using password grant type):

POST /epic/oauth/v2/token HTTP/1.1Host: api.epicgames.devContent-Type: application/x-www-form-urlencodedAuthorization: Basic Zm9vOmJhcg==
grant_type=authorization_code&deployment_id=foo&scope=basic_profile&code=cfd1de1a8d224203b0445fe977838d81&redirect_uri=https://www.example.com

The response contains the following fields:

Response Description
access_token The access token, which may optionally have a prefix (for example: eg1~token). Pass this value "as-is" (do not change it) in the Authorization header using the Bearer type on any requests to EOS services.
expires_in The number of seconds until the token expires.
expires_at The expiration date in ISO 8601 format.
account_id The Epic Games account ID for the user that the token was generated for.
client_id The Client ID which was used to generate this token.
application_id The Application ID which the Client is associated with.
token_type The type of token generated, value will always be bearer.
refresh_token The refresh token might be returned depending on the client configuration. You can use this refresh token to extend a session before or after the access token has expired.
refresh_expires The number of seconds until the refresh token expires.
refresh_expires_at The refresh token expiration date in ISO 8601 format.
